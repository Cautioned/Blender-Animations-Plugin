Privacy Policy

Last updated April 15, 2026

Scope

This policy covers the Blender Animations Plugin itself, as well as the public pages that describe the addon, its privacy policy, and its terms of service.

What the plugin uses Roblox for

The plugin uses Roblox sign-in and asset loading so you can access Roblox assets that your account is allowed to use, including private mesh content.

Data the plugin may handle

Depending on how you use the plugin, it may handle your Roblox sign-in code, state value, access token, refresh token, session key, asset ids, and Roblox responses needed to import assets into Blender.

How credentials are stored

The plugin keeps access tokens, refresh tokens, and session keys in memory for the current Blender session. The addon code says these values are not saved to disk by the plugin. Logging out clears local sign-in state and tries to revoke the refresh token with Roblox.

What we do not do

This project does not run its own account system, does not run its own database for plugin user data, and does not intentionally use ads, tracking, or data resale. Hosting providers and Roblox services may still keep their own logs under their own policies.

Sharing and third parties

To sign in and load protected assets, the plugin sends requests to Roblox. Your browser, internet provider, and Roblox may all process request data under their own terms and privacy policies.

Your choices

You can avoid sign-in entirely by not logging in, and you can clear session state by logging out or closing Blender. If you enter a session key, remove it from the plugin when you no longer want it available in that session.

Contact

For project questions about this policy, use the repository issue tracker or support channel linked from the project repository.
